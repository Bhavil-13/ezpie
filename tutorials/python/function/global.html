Global variables in python

In This article you will get to know about the variables you create outside a function.

What is a global variable?

A global variable is the variable created outside a function. Means that it's the opposite of the local variable.
Therefore it can be access outside and inside a function.

Here we have a variable called x with value 'global', since it's a global variable. Next inside the function do()
we are printing the value of x, which in turn gives us X inside is global. Since the variable x is created outside
the function it was accessable even inside the function.

But This is not true in case of changing the value of x!

Output:

local
X outside is global

Now you may have thought that if the value of x is changed inside the function then way it's giving a wrong
output outside the function?

Will to your suprise the variable x inside the function do() is a diffrent variable from the variable x outside the
function! How you ask? will the variable x outside is a global variable and the variable inside the function is a
local variable!

Try doing this

What output did you get? Somthing like this

File "c:\ezpie-python\main.py", line 8, in <module>
do()
File "c:\ezpie-python\main.py", line 5, in do
x = x + 'local'
UnboundLocalError: local variable 'x' referenced before assignment

This gives an error that's because python takes x inside the function do() a local variable.

To change a global variable inside a function we need to use the global keyword.

What is the global keyword?

The global keyword prevents errors like the one which happened above.

In terms of basic it means that if a variable define outside a function can only be changed inside a function
if it's defined using the global keyword.
