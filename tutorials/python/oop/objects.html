OOP in python

In this article you will get to know about what is called object-oriented programming(OOP)

What is object-oriented?

Python is a object-oriented programming language. Means that rather then functions python also have classes.
To make it simple take classes as a blueprint of a rocket, and with that blueprint you can make more then a 100
rockets right?

The same way take an object as a rocket, now the rocket(object) will have all the properties of the blueprint(class).
More on the detail, object-oriented means that it only focuses on objects.

More on, an object is just a collection of data which are variables and methods which are just functions!
So it means that a variable with functions and more variables!

How to define a class

The same way we define a function using the def keyword, we define a class using the class keyword.

Remember the doc string? Well if yes then, even classes can have doc strings. And like always, it's not mandatory,
but recommended.

Take an example

A class can have a number of attributes, these attributes are variables and functions. There even are special attributes,
for exmaple the __init__(), which is called when ever we call a class in an object.

When we create a class, an object with the same class name gets created. This object helps us to access the attributes
in a class.

Output:

<class '__main__.Person'>
<function Person.hi at 0x000002CE78D9C4C0>

Not what you expacted right? Well it's not python it's infact the print function! Yes we are trying to print a function,
therefore what will you get? a function in hexadicimal!

How to create an object?

Creating an object is really helpfull, even though we can just use the class name but will you repeat the same thing
for two totally diffrent variables? Then it will give the same values.

An object can be used the same way we used directly the class name.

# Create a variable in the same file
tom = Person()

Now we have a variable called tom, this is an inheritance of the Person class. Then we can use any attribute on the
Person class using this variable as prefix.

Methods of this variable are functions of the Person class.

Means that, Person.age is an attribute, therefore tom.age is a method.

Output

Hi! Tom, you are 20 old

Did you notice the self parameter in the attribute hi()? But we only called it without passing any argument.

This is because when we called the method hi() the object itself is passed as an argument! Therefore, tom.hi() will be
Person.hi(tom)!

Basically, if we have a method with random numbers of parameter, and called with an object the object will be passed
as the first argument then the others will come!

Therefore, the first parameter of the method must be the object itself. Therefore, it's called self. Although you can name
it anything you want, but in real life projects(and programmers), it's recommended to name it self

Constructor methods

Any method starting with double underscore and ending in double underscore, is considered as a Constructors

Fo example, the most used constructor __init__(), this is a constructor which is called whenever a new object of that class
is instantiated.

This is called a constructor in object-oriended programming(OOP), it's mustly used for initializing all the variables.

Example

Here we created a class that shows the output as 'Hi Tom, you are 20 years old'. This class has two methods as
__init__() and output().

We have the __init__ method to initialize two variables and output() to display the text after formating it.
